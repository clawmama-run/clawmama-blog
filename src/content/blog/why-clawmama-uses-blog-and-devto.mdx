# Source-first publishing with syndication

The ClawMama blog in this repository is the **canonical source**. That means every post’s
content, links, and metadata are first authored and reviewed here.

Dev.to is used as a distribution channel, not an upstream source.

```markdown
src/content/blog/
  ├── post-one.md
  ├── post-two.mdx
  └── ...
```

## Why this works

- One place to review and version posts
- Cleaner analytics and change history
- Easy republishing when brand messaging shifts

## Suggested workflow

1. Draft/update Markdown or MDX in `src/content/blog`.
2. Run review checks and build locally.
3. Publish the site and push Git updates.
4. Run the sync tool when ready to mirror to Dev.to.

This MDX example keeps the structure the same as Markdown while allowing richer embeds when needed.
